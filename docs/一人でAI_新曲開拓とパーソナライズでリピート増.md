# 一人でAIと対話する利用：新曲開拓・発見とパーソナライズでリピートを増やす

**一人でAIと対話してチャットを利用する場合**、ユーザーが **AI によって新しい曲の開拓・発見** につながる体験を提供することで、サービスの向上と **リピーター増** を目指す。**パーソナライズ** の可能性を追求する方針を整理する。

---

## 1. 前提と目標

| 項目 | 内容 |
|------|------|
| **利用シーン** | パーソナルルームなどで、**一人でAIと対話**しながら曲を聴き、チャットする。 |
| **価値** | AI が **新しい曲の開拓・発見** のきっかけになる → ユーザーにとって「また来たくなる理由」が生まれる。 |
| **目標** | この体験の質を上げることで **サービスの向上** と **リピーターの増加** を実現する。**パーソナライズ** を強く効かせる。 |

→ 一人利用では「誰かと話す」より **「自分に合った曲との出会い」** が主な価値になる。ここをパーソナライズで強化する。

---

## 2. パーソナライズが効く「新曲開拓・発見」の形

### 2.1 何をパーソナライズの材料にするか

| データ | 用途 |
|--------|------|
| **貼った曲・聴いた曲** | session_songs（posted_by_user_id）、参加した会で流れた曲。好みのジャンル・時代・アーティストの傾向。 |
| **発言・リアクション** | 「この曲好き」「サビがいい」など。何を褒めているかで嗜好を補強。 |
| **user_ai_context** | 既存の「傾向・好み・癖」の要約。レコメンドの入力として利用。 |

→ **その人が今までに聴き・貼り・褒めたもの** を把握したうえで、「まだ聴いていないが好みに合いそうな曲」を提案する。

### 2.2 新曲開拓・発見のパターン（パーソナライズ付き）

| パターン | 内容 | パーソナライズの効かせ方 |
|----------|------|----------------------------|
| **「この曲を気に入ったなら、次はこれ」** | 今聴いた曲や貼った曲から、**影響元・同時代・同じ雰囲気** の曲を AI が数本提案。YouTube URL を貼って試聴できる。 | そのユーザーの過去の好み（ジャンル・時代）を渡し、「すでに聴いている曲は除く」「まだ貼っていない曲を優先」する。 |
| **「〇〇さんなら、このアーティストも好きかも」** | 貼った曲・褒めた曲のアーティストから、**関連アーティスト** を提案。 | user_ai_context の「好みの傾向」と組み合わせ、押しつけにならない幅で提案。 |
| **曖昧な「あの曲」探し** | 「昔CMで流れてたピアノの洋楽」など、断片から曲を推論し、候補の YouTube URL を提示。 | 過去にそのユーザーが聴いた年代・ジャンルをヒントに、候補の優先順位を変える。 |
| **テーマで開拓（音楽史の旅など）** | 「ブリットポップの隆盛」「サンプリングの歴史」など、テーマに沿って AI が数曲を順に案内。 | その人がすでに好きなジャンル・時代に合わせてテーマや曲順を調整。初めてのジャンルには入りやすい曲から。 |
| **「今日の一枚」** | その日の気分や「最近聴いていないジャンル」を考慮し、1曲を AI が勧める。 | 直近の聴き方・参加履歴から「飽きていないか」「広げるならどこか」を推測。 |

→ いずれも **一人でAIと対話する** 文脈で、**その人専用の「開拓・発見」** として提供する。

## 3. リピーター増につなげる設計

| 観点 | 内容 |
|------|------|
| **「また聴きたくなる」** | 毎回、**自分に合いそうな新曲や関連曲** を少しずつ提示する。やり尽くした感を出さず、「次に来たらまた何か見つかりそう」と思わせる。 |
| **「進捗が残る」** | 聴いた曲・開拓したテーマ・「気に入った」をつけた曲を **マイページや履歴** で見られるようにする。**続きから再開** できると、一人利用の動機が続く。 |
| **「AIが自分の好みを覚えている」** | 2回目以降、「前回〇〇を聴いてたね。その流れで今度はこれもどう？」のように **継続性** を見せる。パーソナライズが効いていることを実感させる。 |

→ 一人でAIと対話する利用では、**新曲開拓・発見の質** と **パーソナライズの手応え** が、リピートの主な理由になる。

---

## 4. パーソナライズの可能性を追求する方向（具体案）

### 4.1 短期で取りうるもの

- **曲解説のあとに「次に聴くなら」を1〜3本提案**  
  今の曲のアーティスト・ジャンル・時代をもとに、AI が短文で提案し、必要なら YouTube URL を付ける。ユーザーの **貼った曲・聴いた曲の履歴** を渡し、重複・すでに聴いたものを避ける。
- **「この曲好き」と言った曲を記録**  
  発言やリアクションで「好き」と分かった曲を user_ai_context や別テーブルに蓄積し、**好みの精度** を上げる。
- **入室時の「前回は〇〇を聴いていた」**  
  一人利用のパーソナルルーム入室時、直近で聴いた曲やテーマに触れ、「続きでこれもどう？」と提案する。

### 4.2 中期で目指すもの

- **「あなた向けおすすめ」ブロック**  
  パーソナルルームやマイページに、「今日の一枚」「〇〇さんが好きそうな関連曲」を 3〜5 本、YouTube リンク付きで表示。user_ai_context と聴取履歴で更新。
- **連想レコメンドの対話**  
  「Oasis が好きなら Blur / The Beatles のこの曲も」のように、**影響関係・時代・ジャンル** を説明しながら提案。ユーザーが「もっと〇〇系」と言えば、その方向に絞る。
- **テーマ別「音楽の旅」のパーソナライズ**  
  テーマは固定でも、**最初の曲や順番** をその人の好みに合わせる。「ブリットポップの旅」なら、すでに好きなアーティストから始める、など。

### 4.3 長期で追求するもの

- **「開拓の履歴」の可視化**  
  いつ・どんな曲を「開拓」したか、どのテーマを聴いたかをタイムラインやマップで表示。**自分の洋楽の広がり** が実感でき、リピートの動機になる。
- **嗜好の変化の取り込み**  
  長期的な聴取履歴から「最近は〇〇が増えている」などを要約し、user_ai_context を更新。提案の幅を **時間とともに最適化** する。
- **プレイリスト Export**  
  一人利用で「気に入った」にした曲を、Spotify / YouTube のプレイリストとして書き出し。**他サービスとの接続** と「ここで発見した曲を日常で聴く」循環を作る。

## 5. データ・テーブルとの対応（方針）

| 用途 | 利用するデータ・テーブル |
|------|---------------------------|
| 好みの把握 | chat_messages（発言）, session_songs（貼った曲・聴いた曲）, リアクション（任意テーブル） |
| 要約・文脈 | user_ai_context（傾向・好み・癖） |
| 提案の重複回避 | そのユーザーがすでに貼った・聴いた session_songs の一覧 |
| 「気に入った」の蓄積 | 既存の session_songs に「いいね」フラグ、または user_favorite_tracks のようなテーブル（任意） |

→ 既存の **登録ユーザー管理・user_ai_context** を、**一人利用の新曲開拓・パーソナライズ** でも中核として使う。

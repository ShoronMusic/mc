# 初回ユーザーご意見回収プラン（報酬なし）

最終更新: 2026-04-14

## 目的

- 初回利用者の率直な体験評価を短時間で回収する
- 離脱理由・詰まり箇所を特定し、再訪率改善につなげる

## KPI（目安）

- 初回ユーザー回答率: 20〜35%
- 自由記述率: 8〜15%
- 初回→2回目再訪率: 改善傾向を追跡

## 回収設計（最小30秒）

1. 退室時にワンタップ評価を表示
2. 必要最小限の追加質問を表示
3. 任意の自由記述を受ける

## 5段階評価（追加）

初回ユーザー向けモーダルの先頭に、以下の質問を配置する。

- 質問文:
  - 「このチャットサイト全体の印象を、-2（とても悪い）〜 +2（とても良い）でお選びください。」
- 評価値:
  - `-2` / `-1` / `0` / `+1` / `+2`
- UI方針:
  - 5つのボタンを横並び表示
  - 初期値は未選択
  - 選択後に枠色を変えて選択状態を明示

## 質問フロー（推奨）

### Q1（必須）

- 5段階評価: `-2` / `-1` / `0` / `+1` / `+2`

### Q2（任意、複数選択）

- 「どこで迷いましたか？」
- 選択肢例:
  - 入室方法
  - YouTube URL貼り付け
  - AIへの質問方法
  - 画面の見方
  - 特になし

### Q3（任意、自由記述）

- 「改善してほしい点があれば教えてください」

## 表示タイミング

- β版のあいだは「退室時アンケート」のみ表示対象にする
- 初回セッションの退室操作時にモーダル表示
- スキップ可能（同セッションで再表示しない）
- 回答済みユーザーには再表示しない

## 頻度制御（しつこくしない運用）

- 初回のみ優先表示（回答またはスキップで1回完了）
- 2回目以降は最終表示から7日未満なら非表示
- β版のあいだは月間表示回数は最大2回まで
- 一度回答したユーザーには30〜60日再表示しない
- 低評価（`-2` / `-1`）時のみ任意の自由記述入力を追加表示

### 推奨判定ロジック（例）

- 表示する条件:
  - `is_first_session = true`
  - または `last_shown_at から 7日以上` かつ `monthly_show_count < 2` かつ `last_answered_at から 30日以上`
- 表示しない条件:
  - 当セッションですでにスキップ済み
  - 回答済みかつクールダウン期間内

### クライアント保持項目（例）

- `feedback_last_shown_at`
- `feedback_last_answered_at`
- `feedback_show_count_month`
- `feedback_last_score`（任意）

## 収集データ項目（最小）

- `rating_score`（-2〜+2）
- `pain_points`（複数選択）
- `free_comment`（任意）
- `room_id`（任意）
- `is_first_session`（true/false）
- `created_at`

## 運用ルール

- 週1回、回答を15分で確認
- 同一課題が週3件以上で改善候補に昇格
- 重大な不具合系コメントは即時切り分け

## 改善の見せ方（回答率向上）

- トップまたはガイドに「最近の改善」欄を短文で掲載
- 例: 「ご意見を受けて、初回案内を改善しました」

## 実施ステップ（2週間）

- Week 1: モーダル実装、保存API接続、集計軸定義
- Week 2: 上位課題を1〜2件改修し、改善内容を表示

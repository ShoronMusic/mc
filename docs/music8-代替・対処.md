# Music8 JSON が無い・一致しないときの代替・対処

視聴履歴の **メインアーティスト** / **ソングデータ** タブは、  
`https://xs867261.xsrv.jp/data/data/` 配下の **Music8 と同じスラッグ規則** で JSON を探します。  
ヒットしない場合の**代替の進め方**です。

---

## 1. まずアプリ側で確認すること

| 確認 | 内容 |
|------|------|
| **タイトル形式** | YouTube は `アーティスト - 曲名 (Remastered 等)` になっていると、曲名は **ハイフン以降**だけを検索に使います（実装済み）。 |
| **複数アーティスト** | `feat.` などで列挙された名前（例: Stevie Wonder）でも **順に検索**します（実装済み）。 |
| **メインアーティスト JSON** | `artists/{スラッグ}.json` … スラッグは **The を除きスペース→ハイフン・小文字**（例: `Gary Byrd` → `gary-byrd`）。 |

ブラウザで直接開いて 404 かどうか見ると早いです。

- アーティスト:  
  `https://xs867261.xsrv.jp/data/data/artists/{スラッグ}.json`
- 曲（直リンク）:  
  `https://xs867261.xsrv.jp/data/data/songs/{アーティストスラッグ}_{曲スラッグ}.json`  
  例: `gary-byrd_the-crown.json`

---

## 2. 代替①：Music8 側にデータを足す（本命）

運用で Music8（WordPress / 同サーバの data）を更新できる場合:

1. **アーティストページ**に該当アーティストが存在するか確認。  
   - 無ければ **アーティストを新規登録** → `artists/{slug}.json` が生成される想定。
2. **曲**をそのアーティストに紐づけて登録。  
   - **スラッグ**がアプリの `songTitleToMusic8Slug` と一致しているか（英数字・ハイフン、括弧内は除去後の曲名）。
3. 曲だけ直 JSON で持たせる場合は  
   `songs/{artistSlug}_{titleSlug}.json` が実在する形にする。

※ Music8 の管理画面・手順は **あなたの Music8 運用ドキュメント** に従ってください。

---

## 3. 代替②：別アーティスト名で Music8 に載っている場合

コラボ曲は **片方のアーティストページにしか曲が無い**ことがあります。

- アプリは **メインアーティスト（DB）→ タイトルから拾った feat. 側**の順で検索します。
- まだ無い場合は、**Music8 上で「どちらのアーティストのディスコグラフィに載せるか」**を揃えるか、**両方のページに同じ曲を登録**するとヒットしやすくなります。

---

## 4. 代替③：コード・データ源を変える（開発向け）

Music8 を使わない／補完したい場合の例:

| 方針 | 内容 |
|------|------|
| **自前テーブル** | Supabase に `song_external_facts` などを用意し、`video_id` キーで表示（Music8 より優先表示も可）。 |
| **別 API** | Wikipedia / MusicBrainz / 自前 CMS などを **別タブ or 同タブ内でフォールバック**（要実装）。 |
| **手入力リンク** | 管理画面から「この video_id の解説 URL」を保存して表示するだけにする。 |

必要なら、どのデータ源を優先するか決めたうえで実装方針を切ります。

---

## 5. ユーザー向けの見せ方（運用）

- データが無いときは **「music8 にこの曲のデータがありません」** と出ます（仕様どおり）。
- よく流す曲は **Music8 に先に登録**しておくと、タブが意味を持ちます。

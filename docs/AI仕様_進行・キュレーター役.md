# AI 仕様：進行・キュレーター役

AI の役割は次の **2つ** を想定する。

1. **曲情報の提供** … YouTube の曲の「アーティスト」と「タイトル」を元に、豆知識・解説を提供する。
2. **チャットへの参加** … 人間の発言への評価・感想などを短く返し、程よく会話に参加して会を盛り上げる。

---

## 1. 役割①：曲情報の提供（入力と出力）

| 項目 | 内容 |
|------|------|
| **入力** | 曲の **アーティスト名（artist）** と **曲タイトル（title）** |
| **出力** | 曲の豆知識・解説（140〜240文字程度）。進行コメントやキュレーション文。 |

- YouTube URL からは **videoId しか取れない** ため、**アーティストとタイトルは別途取得する**必要がある。
- 取得方法: oEmbed の `title` / `author_name` のパース、または YouTube Data API の `snippet.title` / `snippet.channelTitle` をそのまま／AI で構造化して **artist / title** に分ける。

---

## 2. データの流れ

```
ユーザーが YouTube URL を貼る
  ↓
videoId を抽出し、oEmbed または Data API でメタデータ取得
  ↓
「アーティスト」「曲タイトル」を決定
  （例: title を "Artist - Track" でパース / チャンネル名を artist に / AI で分離）
  ↓
session_songs に youtube_video_id, artist_name, track_title を保存
  ↓
AI（Gemini）に (artist_name, track_title) を渡して曲情報・解説を生成
  ↓
生成テキストをチャットに投稿（AI 発言）＆ Now Playing エリアに表示
  ↓
session_songs.ai_commentary に保存（任意）
```

→ **AI が参照するのは「アーティストとタイトル」のみ。** videoId や URL はプレイヤー用で、AI の解説生成の入力としては使わない。

---

## 3. 役割②：チャットへの参加（会話参加・盛り上げ）

AI は **チャットの会話にも参加** する。人間の発言への評価・感想などを返し、**程よく** 会話に加わり、適度に会を盛り上げる役割を持つ。

### 3.1 やること

| 内容 | 説明 |
|------|------|
| **相槌・共感** | 「それわかります！」「このサビ、本当にいいですよね」など、短い一言。 |
| **評価・感想** | 参加者の意見や感想に対して、AI も自分の「聴き手としての感想」を1〜2文で返す。 |
| **話題を繋ぐ** | 沈黙しそうなときや、誰かの発言を拾って「〇〇さん言う通り、このドラムの質感とか…」と話題を振る。 |
| **盛り上げ** | 盛り上がっているときは乗る。静かなときは押しすぎず、さりげなく一言入れる。 |

### 3.2 やらないこと・注意

| 項目 | 内容 |
|------|------|
| **しゃべりすぎない** | 人間の会話の主役は人間。AI は「ちょうどいい常連さん」程度の存在感に留める。 |
| **解説モードに偏らない** | 曲解説は「曲が変わったとき」に集中。普段のチャットでは「感想・相槌」を優先。 |
| **否定しない** | 参加者の趣味や解釈を否定せず、肯定的に受け止める。 |
| **長文にしない** | チャット参加時の発言は **1〜2文・短く**。長い解説は曲情報エリアに任せる。 |

### 3.3 トリガー（いつ発言するか）

- **誰かが発言した直後** … 内容に応じて相槌・感想を1回返す（毎発言に反応すると多すぎるため、N 発言に1回など間引くルールを検討）。
- **沈黙が続いたとき** … プラン通り「30秒以上発言なし」などで、曲や雰囲気に合った一言を投稿。
- **メンション・質問されたとき** … 「@AI この曲誰だっけ？」などには必ず短く回答。

### 3.4 トーン

- **親しみやすい音楽仲間**。敬語でもタメ口でもよいが、統一する（例: です・ます調でやわらかく）。
- 知識をひけらかさず、「一緒に聴いている仲間」としての感想・共感を心がける。

### 3.5 実装メモ

- **入力**: 直近のチャットログ（発言者・本文・時刻）＋ 現在流れている曲（artist / title）。
- **出力**: 1〜2文の短文。`chat_messages` に `message_type='ai'` で保存し、チャット欄に表示。
- **レート制限**: 曲解説と会話参加で Gemini を叩く回数が増えるため、会話参加は「間引き」や「沈黙時のみ」などでコスト・過多投稿を抑える。

---

## 4. DB との対応（曲情報）

**session_songs** では、AI の入力元となる項目を「必須に近い形」で持つ。

| カラム | 用途 |
|--------|------|
| **artist_name** | AI への入力の一つ。表示・振り返り・ユーザー館でも使用。 |
| **track_title** | AI への入力の一つ。表示・振り返り・ユーザー館でも使用。 |
| youtube_video_id | 再生・サムネイル・リンク用。AI の解説生成には使わない。 |
| youtube_title | 取得した生タイトル（フォールバック・表示用）。 |
| ai_commentary | AI が生成した曲情報・解説文。 |

- アーティスト・タイトルが取れない場合（oEmbed 失敗など）は、**youtube_title だけを AI に渡す**などのフォールバックを用意するとよい。

---

## 5. AI へのプロンプト方針（曲解説の例）

- **入力**: `artist_name`, `track_title`（取得できない場合は `youtube_title` のみ）。
- **役割**: 進行・キュレーター。洋楽好きのリスナーに寄り添う「ちょうどいい」豆知識。
- **形式**: 140〜240文字。専門用語は少なめ。事実ベースの解説＋一言感想。
- **禁止**: 歌詞の全文掲載、過度に長い説明、上から目線の解説。

例（プロンプトに渡すテキスト）:
「アーティスト: Oasis / 曲名: Whatever について、洋楽ファン向けに140〜240文字で豆知識を1つ書いてください。」

---

## 6. まとめ

- **役割①（曲情報）**: 曲の「アーティスト」と「タイトル」を入力に、豆知識・解説を提供する。YouTube のメタから artist_name / track_title を導出し、AI に渡す。
- **役割②（チャット参加）**: 人間の発言への評価・感想・相槌を短く返し、程よく会話に参加して会を盛り上げる。しゃべりすぎず、主役は人間に。

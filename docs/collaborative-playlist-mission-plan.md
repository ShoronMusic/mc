# 協力型プレイリスト・ミッション — 単独企画メモ

## 0. 位置づけ

- 洋楽サロンにおける **「テーマ付きで皆が選曲に参加する」** 体験と、**一人でもミッション感覚で曲を集められる** 体験を、**サイト側（＋AI）からお題を提案**できる形で進めたい、という企画の置き場。
- ルーム内ミニゲーム総論は `docs/room-social-music-games-plan.md` の **§3.6 協力型プレイリスト・ミッション** と接続するが、本ファイルは **カレンダー・文化コンテキスト・ソロ進捗・AI提案** まで含めて単独で深掘りする。
- 実装タスク一覧ではなく、**プロダクト意図・テーマ設計・段階的導入**のメモ。仕様確定時は別途 API/UI 仕様に落とす。

---

## 1. 狙い（何を良しとするか）

| 観点 | 内容 |
|------|------|
| 参加のしやすさ | 正解・順位付けより **「このお題なら1曲貼れる」** を主軸にする。 |
| リピート | **期間限定お題**で「今週だけの会」が立ち上がりやすくする。 |
| 一人利用 | 部屋に誰もいなくても **お題に沿って10曲集める** など、**ゲーム感・コレクション感**で滞在時間と愛着を伸ばす。 |
| AI の役割 | **お題の提案・言い換え・季節の候補**に寄せる。採点や「正しい洋楽」の判定は持たない（揉め・幻覚リスクを避ける）。 |
| 既存体験との関係 | 選曲・チャット・（将来）マイリストや履歴と **同じ YouTube 貼り付け** で完結させたい。専用ゲームサーバーは不要な方向を優先。 |

## 2. テーマ（お題）のタイプ案

### 2.1 一定期間のイベント系（カレンダー・期間固定）

- **クリスマス**、**ハロウィン**、**バレンタイン**、**大晦日〜年始** など。
- 運用上は **開始日・終了日** をサイト側で持ち、その期間だけ UI に「今週のお題」として出す想定。
- 期間が被る場合は **優先ルール**（例: ハロウィン終了後に秋深まり、など）を後で定義。

### 2.2 記念日・「●●の日」（当日および前後日）

- 世界的な記念日に加え、**日本で話題になりやすい日**（音楽・文化系）をカレンダーに載せる案。
- **実施ウィンドウ**: 「当日のみ」より **前日〜翌日（±1）** まで有効、など緩めにすると参加機会が増える。
- 出典が曖昧な「●●の日」は **断定せずラベル**（「今日は〇〇にちなんだお題です」程度）に留める選択肢あり。

### 2.3 日本独自の季節・風習・文脈（マイクロシーズン）

例（ラベル案。最終リストは運用で調整）：

| 文脈 | お題の方向性（例） |
|------|-------------------|
| **6月・梅雨** | じめじめした夜に合う、雨の窓辺、室内で聴きたい、などムード寄り |
| **7〜8月・夏祭り・夏休み** | 夏の夜、花火、ドライブ、フェス気分（ジャンル固定は避け、**気分語**で） |
| **お彼岸・秋** | 夕暮れ、帰路、少し寂しさのある曲、など**押し付けない**トーン |
| **春の入学・卒業・別れ** | 新しい始まり／旅立ちのイメージ（歌詞に依存しすぎない案も併記可） |

※宗教・地域イベントは **押し付けがましくならない** 文言と **スキップ可能** を前提にする。

### 2.4 用途・シチュエーション（時間帯・ライフスタイル）

- **お酒に合う曲**（酩酊・未成年を連想させすぎない表現に注意）
- **目覚めの1曲**
- **睡眠導入**（激しい曲も「逆説で」と言わず、**穏やか寄りを推奨**する程度のガイド）
- **帰り道・通勤・作業用** など、**シーン語彙**で統一しやすい。

### 2.5 聴き方の切り口（日本人リスナー向け・洋楽）

- **歌詞より音・フィーリング重視** と明示したお題を用意する。
- 例ラベル: 「日本語が分からなくても気持ちいい」「イントロで決める」「ギターの音の気持ちよさ」「ドラムのノリだけで選ぶ」など。
- AI 文案では **「正解の聴き方はない」** を添え、マウント文化を避ける。

### 2.6 その他・拡張

- **洋楽の特定ジャンル縛り**は強いとハードルが上がるため、初期は **ムード・季節・シーン** を主軸にし、ジャンルは「おまけタグ」程度から始める案。

## 3. 楽しみ方パターン（協力 / ソロ）

### 3.1 協力型（部屋内）

- 現在の **1曲ずつ選曲・同期視聴** の延長で、**「今週のミッション：テーマ X」** を部屋または全体に表示。
- 各自 **1曲ずつ（または上限 N 曲）** を貼り、会話は「なぜその曲か」を短く言う程度までを推奨（強制はしない）。
- **達成条件の例**: 「部屋内でテーマに沿った選曲が合計 10 本に達したら」バッジ表示、など**ソフトな可視化**から始めると安全。

### 3.2 ソロ型（一人＋AI）

- **お題に対して10曲集める**（マイページのマイリスト、または将来の「ミッション用キュー」に紐づけ）。
- **進捗バー**（3/10…）と **AI からの軽い相槌**（「次はこんなムードもありですか？」程度）のみに留め、**採点・却下はしない**。
- 既存の **履歴・お気に入り** と組み合わせ、「履歴からこのお題に近い曲を提案」は **後段**（プライバシー・説明文の整備が必要）。

## 4. サイト側（＋AI）で「提案」するときの形

### 4.1 提案チャネル（段階案）

| 段階 | 内容 | メリット / 注意 |
|------|------|----------------|
| **A. 静的＋日付** | サーバーまたは設定ファイルの **テーマカレンダー** を日付で切り替え、UI に1行表示 | AI コストゼロ。運用で差し替え可能。 |
| **B. テンプレ＋軽い生成** | 固定テンプレに **季節・天気（任意）** を差し込む短文のみ AI | コスト小。幻覚リスクを限定。 |
| **C. フル生成** | お題文を毎回 AI が書く | 多様だが **品質・トーン事故** の監視が必要。 |

**推奨の進め方**: まず **A** でカレンダーと UI の置き場を作り、反応を見て **B** に拡張。C は必要になってから。

### 4.2 AI に任せる範囲（原則）

- **任せる**: お題の言い換え、**複数候補から1つ選ぶ理由の短文**、**その日の文脈に合わせた補助説明**（断定を避ける）。
- **任せない（初期）**: 「この曲はお題に合わない」判定、ランキング、他者への評価コピー。

### 4.3 オフ・スキップ

- 部屋オーナーまたはユーザー設定で **お題バナーを非表示** にできる余地を残す（既存のオーナー向けトグル文化と整合）。

## 5. データ・UI のたたき（仕様化前のメモ）

- **エンティティ案**: `MissionTheme`（id, 表示名, 説明文, 開始日時, 終了日時, タグ[], 対象ロケール `ja` など）。
- **参加の記録**: 最初は **クライアント表示のみ** でも可。永続化するなら `user_id` × `theme_id` × `video_id` のログは **後から**。
- **部屋との関係**: 「グローバルお題」と「部屋で上書きお題」のどちらから始めるかは未決。初期は **グローバル1本** が実装しやすい。

---

## 6. リスク・コンプライアンス（先に書いておくこと）

- **アルコール・夜遅い時間帯**のコピーは、未成年利用・過度な飲酒推奨に見えない表現にする。
- **宗教・国民的行事**は任意参加・中立表現。
- **著作権**: お題文で **歌詞の長文引用**をしない。曲の貼り付けは既存どおり YouTube リンク。
- **幻覚**: 記念日の「由来」を AI に書かせる場合は **出典フラグ**または **固定文の人間監修**を検討。

---

## 7. MVP の切り方（提案）

1. **テーママスタ**（JSON または DB）＋ **日付で1件選択**する API または静的設定。
2. ルーム（またはトップ）に **「今週のお題」バナー** ＋ **短い説明文**（人間監修テンプレ）。
3. **ソロ10曲**は **マイリストの手動運用**でも再現できる案内文から始め、**進捗 UI** は次フェーズ。
4. 協力型の「合計 N 本」は **Ably/集計の負荷**を見てから。まずは **チャット上の自発カウント**でも検証可。

## 実装（ソロ・10曲・AIコメント＋ルーム連携）

- **ルーム（主導線）**: マイページで進行中のお題ミッションがあるとき、チャット **ヘッダー2段目**（「チャット／チャットサマリー…」の直下）に **お題・ラベル・(n/10)・実施中** を固定表示。発言欄の **送信の上に「お題曲送信（β）」** も出る。URL を貼って **お題曲送信** を押すと `themeId` が付き、**通常の AI 曲解説のあと**に **【お題講評】**（`POST /api/user/theme-playlist-mission/room-blurb`）。通常の「送信」での URL 選曲はお題に紐づけない。進捗の再取得は `room-blurb` 成功時・マイページでの追加／開始時のクライアントイベント（`theme-playlist-mission-client-events`）。
- **マイページ**タブ「お題プレイリスト」: 固定お題を表形式で一覧し、ミッション選択・進捗・エントリを表示。**一旦解除（paused）** にしても途中曲は保持され、同じお題の「開始/再開」で続きから再開。エントリには **選曲者名（`selector_display_name`）** と **日時（`created_at`）** を保持・表示。
- **API**: `GET`/`POST` `/api/user/theme-playlist-mission`（`action=pause` で一旦解除）、`POST` `/api/user/theme-playlist-mission/entry`（マイページからの追加）、`POST` `/api/user/theme-playlist-mission/room-blurb`（ルーム選曲＋お題時の講評・記録）
- **UI 部品**: `ThemePlaylistMissionPanel.tsx`（マイページ）、`Chat.tsx` の `themePlaylistActiveMission`、`ChatInput` の `themePlaylistRoomSubmit` と「お題曲送信」経由の `onVideoUrl(..., { themePlaylistThemeId })`、`useThemePlaylistRoomSubmitMission`
- **テーマ定義**: `src/lib/theme-playlist-definitions.ts`
- **Gemini**: `theme_playlist_comment`（`src/lib/theme-playlist-ai-blurb.ts`）
- **DB**: `docs/supabase-setup.md` 第 18 章

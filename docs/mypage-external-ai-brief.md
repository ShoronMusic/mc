# マイページ機能概要（外部 AI 向けブリーフ）

本書は **musicaichat** の「マイページ」が何をする UI かを、別モデル・外部ツールに渡すための要約である。実装の一次情報は `src/components/mypage/MyPage.tsx`（部屋内モーダル）。認証は Supabase（Google 等）。秘密情報・`.env` は記載しない。**入室前の参加方法**は §1 を参照。

---

## 1. 参加方法（ユーザー登録 2 種類とゲスト参加）

部屋 URL（`/{roomId}`）で入室する場合、同意画面のあと **`JoinGate`** が開催可否を確認し、未ログインなら **`JoinChoice`** で参加方法を選ぶ（`src/components/auth/JoinGate.tsx`, `JoinChoice.tsx`）。トップページからの導線（`TopPageLoginEntry.tsx` 等）は別レイアウトだが、認証の系統は同じ Supabase クライアントに依存する。

### 1.1 登録ユーザー（2 種類）

| 方式 | 概要 | 実装の要点 |
|------|------|------------|
| **Google でログイン** | OAuth。**Supabase** `signInWithOAuth({ provider: 'google' })`。リダイレクトはアプリ origin＋`/auth/callback?next=…`（戻り先はクッキー `oauth-return-path` 等で保持）。 | `JoinChoice` の Google ボタン。`NEXT_PUBLIC_SUPABASE_URL` / `NEXT_PUBLIC_SUPABASE_ANON_KEY` が無いとボタンは出ず案内のみ。 |
| **メールアドレスでログイン** | **メール＋パスワード**の簡易会員（ログイン／新規登録の切替、表示名、パスワード忘れで `resetPasswordForEmail` → `/auth/recover-callback`）。Supabase のメール確認が有効なプロジェクトでは **登録直後はセッションが無く**、確認メール後に「すでに登録済みの方はログイン」へ誘導する UI 文言あり。 | `SimpleAuthForm.tsx`（`JoinChoice` から表示）。 |

いずれも成功後は **Supabase セッション付きの登録ユーザー**として部屋に入り、`JoinGate` 側でゲスト用 `sessionStorage` フラグはクリアされる。

### 1.2 ゲスト参加

| 項目 | 内容 |
|------|------|
| **概要** | **アカウントを作らず**ハンドル名（未入力時は「ゲスト」＋自動番号等）だけで入室。ブラウザの **`sessionStorage`** に `mc:guest` / `mc:guest_name` / `mc:guest_room` を保存し、同一ブラウザ・同一部屋なら再訪時に復帰しやすい。 |
| **機能差** | マイページは**限定的**（表示名・選曲参加・効果音・テキスト色など）。お気に入り・貼曲履歴の永続化・フルマイページは**ログイン利用**が前提（`JoinChoice` 上の案内文言あり）。 |
| **部屋内** | 「ユーザー登録」を促すモーダル等あり（`GuestRegisterPromptModal.tsx`）。 |
| **トップのゲスト導線** | `TopPageLoginEntry` の「ゲストで参加」は **体験用部屋 ID** への振り分け（在室が少ない部屋優先など）で `location.href` 遷移。主催者フローはログインまたはゲスト制限の注記あり。 |

### 1.3 前提・フォールバック

- **Supabase 未構成**（anon URL/キーなし）: Google／メールの導線は表示されず、**ゲストのみ**が選べる状態になる（`isSupabaseConfigured()`）。
- **開催・鍵・主催者先行**: 参加方法を選んだあとも、`runRoomEntryGateCheck` で拒否され得る。詳細は `docs/room-playback-chat-summary-features.md` §7・§8。

## 2. 目的と前提

- **場所**: 部屋 UI の「マイページ」ボタンから開くモーダル（専用ルートページではない）。
- **利用者**: **ログイン済みユーザー**がフル機能。**ゲスト**は限定的な別 UI。
- **タブ構成**（ログイン時）: **オーナー機能** | **ユーザー機能** | **曲管理** | **マイリスト**  
  ゲストはタブなし（表示名・選曲参加・効果音・テキスト色＋条件付きで部屋編集ブロックのみ）。

---

## 3. タブ表示条件（オーナー機能）

- **オーナー機能**タブが出るのは次のいずれかを満たすとき:
  - **開催中ライブの主催者**（API `room-live-status` で `isOrganizer` が真）かつ部屋 ID・クライアント IDが取れる、または
  - **チャットオーナー**で、譲渡・AI 雑談停止・comment pack・邦楽解禁・強制退出・選曲モード切替・5 分制限などの**オーナー専用コールバック**のいずれかが親から渡されている場合。
- **主催者のみ**（チャットオーナーでない）: 主に **部屋の表示タイトル・PR（ロビー文）** などの編集（`LobbyMessageOwnerBlock`）。
- **チャットオーナー**: 上記に加え、下記「オーナー機能」の各種トグル・参加者操作。

---

## 4. オーナー機能（要約）

| 領域 | 内容 |
|------|------|
| 部屋プロフィール | 主催者／オーナーが **部屋の名前（表示タイトル）**・**PR 文（ロビー）** を保存（サービスロール経由 API。DB 手順は `docs/supabase-setup.md` 該当章）。 |
| 一曲 5 分制限 | ON 時は長尺 PV でも途中で次選曲へ促す系の挙動（親コンポーネントと連動）。 |
| AI 自由発言 | 沈黙時の AI 雑談の停止／再開。 |
| 曲紹介コメント | 選曲後に出す AI 解説スロット（基本情報・チャート系・歌詞・サウンド・アーティスト情報など）の個別 ON/OFF・プリセット。 |
| 邦楽 AI 解説 | 既定 OFF。ON で邦楽向け AI 解説を解禁。 |
| 参加者 | **オーナー譲渡**（在室者のみ）、**視聴専用／選曲参加**の切替、**強制退出**。 |

---

## 5. ユーザー機能（要約）

| 領域 | 内容 |
|------|------|
| アカウント | 表示名・メールの変更、**アカウント削除**（確認ダイアログ。サーバ側で Service Role 等が必要な場合あり `docs/supabase-setup.md` 7 章）。 |
| 他ユーザー向けプロフィール | 一言・好きなアーティスト枠・補足。**公開はオプトイン**（`visible_in_rooms`）。他参加者への表示はドキュメント上「部屋 UI 接続は別途」の旨あり。 |
| AI 向け趣向メモ | 登録ユーザーの短文メモ。部屋で **「@」付き AI チャット**のプロンプトに**参考**として載る（他ユーザー非表示）。**履歴から自動要約**ボタンで、チャットログ・選曲履歴・お気に入り・マイリスト等を集約した別フィールドを Gemini で更新し、手動メモと併用（`docs/supabase-setup.md` 14・15 章、`user-ai-taste-context.ts` 系）。 |
| 参加体験 | **選曲に参加／視聴専用**、**入室・退室の効果音**（ブラウザローカル優先、部屋連動時は親から制御）、**自分のステータス**（離席・ROM 等）、**チャット発言のテキスト色**。 |

**ゲスト（マイページ（ゲスト））**: 表示名、選曲参加、効果音、テキスト色。主催者条件を満たせば部屋管理ブロックのみログイン版に近い。

## 6. 曲管理タブ（要約）

| サブタブ | 内容 |
|----------|------|
| 選曲リスト | ユーザーがチャットで**貼った（選曲した）YouTube** の履歴。日付・部屋・時刻・ラウンド等。スタイル／年代タグが付く場合あり。ページ送り。**TEXT エクスポート**可。 |
| お気に入り | ユーザーが星等で保存した動画一覧。タグ・ページ送り。**TEXT エクスポート**可。 |
| 参加履歴 | ログイン状態で入室した**会（ギャザリング）**ごとの入室・退出・滞在時間。会タイトル・入室時表示名など。TEXT 保存は UI 上未対応の旨あり。 |

永続化には Supabase テーブルが必要（履歴: `docs/supabase-song-history-table.md`、お気に入り: `docs/supabase-user-favorites-table.md`、参加履歴: `docs/supabase-setup.md` 参加履歴章）。

---

## 7. マイリストタブ（要約）

- **チャットとは独立した個人ライブラリ**（同一 `video_id` は 1 件まで）。
- **新規追加曲**: YouTube URL（または ID）で追加。曲名・アーティスト・メモ・スタイル・年代の編集・削除。**部屋へ選曲**ボタンで発言欄相当の動線に載せる。
- **保存済アーティスト**: マイリストから集約したアーティスト一覧、頭文字フィルタ、曲一覧展開、**PROFILE** でアーティスト情報パネル（`MainArtistTabPanel`）。
- スキーマ・API: `docs/supabase-user-my-list-table.md`、`src/app/api/my-list/route.ts`。より広い設計は `docs/my-list-spec.md`。

---

## 8. 外部 AI が誤解しやすい点（注意）

1. **マイリスト ≠ 貼った曲履歴**: チャットで貼っただけではマイリストは増えない（明示保存・マイページ URL 追加のみ）。貼った曲は「曲管理 → 選曲リスト」。
2. **趣向メモ・自動要約**は主に **`@` チャット**向け。入室挨拶や tidbit に常時載るとは限らない（実装は `fetchUserTasteContextForChat` 等を参照）。
3. **公開プロフィール**はオプトイン。オフの間は他ユーザーが DB から読めない設計を前提にした説明をすべき。
4. **オーナー機能**は「主催者」と「チャットオーナー」で**出る項目が違う**。

# ユーザー向けガイド作成時のメモ（利用手順で説明すること）

後々ユーザー用のガイドページを作る際に、利用手順を示すうえで説明・記載することを忘れないようにメモしておく。

---

## Google認証でログインするときの表示について

- **事象**  
  アカウント選択画面で「**〇〇〇〇〇〇〇〇.supabase.co に移動**」のように、英数字の羅列のドメイン名が表示される。
- **説明すべき内容**  
  - このドメイン（例: `ofxsvhiygmdkwqmaeoid.supabase.co`）は、認証基盤（Supabase）が自動で割り当てている**本番用の公式ドメイン**です。
  - 不審なサイトではなく、ログイン処理を安全に代行しているサービス名として表示されています。
  - サービス名やアプリ名を自分で付けていない場合は、このような英数字のドメイン名がそのまま表示されますが、**そのまま「続行」して問題ありません**。
- **ガイドでの記載例**  
  「Googleでログインを選ぶと、『〇〇〇〇.supabase.co に移動』と表示されることがあります。これは認証を担当するシステムの名前で、安全な画面です。そのままアカウントを選んで続行してください。」

### 同意画面（「次へ」の許可確認）でも同じドメインが出る

- **事象**  
  アカウント選択のあと、**「〇〇〇〇.supabase.co にログイン」**「Googleは、あなたに関する情報へのアクセスを 〇〇〇〇.supabase.co に許可します」といった同意画面でも、同じ英数字のドメインが表示される。プライバシーポリシー・利用規約のリンク先もそのドメインとして表示される。
- **説明すべき内容**  
  - 上記と同じく、認証基盤（Supabase）の公式ドメインであり、不審なサイトではない。
  - **「次へ」を押して許可して問題ありません。** 名前・メールアドレス等はこのサービス内での表示名・識別にのみ使われます。
- **ガイドでの記載例**  
  「次の画面で『〇〇〇〇.supabase.co にログイン』『〇〇〇〇.supabase.co に許可します』と出ます。同じく認証システムの名前です。そのまま『次へ』を押してかまいません。」

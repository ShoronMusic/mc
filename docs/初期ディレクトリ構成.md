# 初期段階のディレクトリ構成

洋楽AIチャット（Next.js）の構築開始時に揃えるディレクトリ構成。MVP 〜 複数ルーム・認証・個人化を見据えた形。

---

## 1. 全体構成

```
e:\mc\
├── app/                    # Next.js App Router
│   ├── layout.tsx
│   ├── page.tsx
│   ├── globals.css
│   ├── (auth)/              # 認証まわり（ログイン・登録）※後で追加可
│   ├── room/
│   │   └── [roomId]/
│   │       └── page.tsx     # ルーム画面（複数ルーム対応時）
│   └── api/                 # API Routes（Gemini 呼び出し、oEmbed 等）
│       ├── ai/
│       └── youtube/
├── src/
│   ├── components/          # UI コンポーネント
│   │   ├── chat/
│   │   ├── player/
│   │   ├── room/
│   │   └── ui/              # 共通 UI（shadcn 等）
│   ├── lib/                 # ユーティリティ・クライアント
│   │   ├── supabase.ts
│   │   ├── ably.ts          # または socket.ts
│   │   ├── youtube.ts
│   │   └── ai.ts            # Gemini 呼び出し（サーバー用は app/api から）
│   ├── hooks/
│   ├── types/               # 共通型定義
│   └── stores/              # 状態（必要なら）
├── server/                  # Socket.io を使う場合の別プロセス（任意）
│   ├── index.js
│   └── package.json
├── public/
├── docs/                    # 企画・仕様（既存）
├── .env.local
├── .env.example
├── next.config.js
├── tailwind.config.ts
├── tsconfig.json
└── package.json
```

---

## 2. ディレクトリの役割

| パス | 役割 |
|------|------|
| **app/** | ページ・レイアウト・API Routes。`page.tsx` はトップ（ルーム一覧 or 1ルーム直行）。`room/[roomId]` は複数ルーム対応時。 |
| **app/api/** | Gemini 解説・相槌、YouTube oEmbed 取得など、サーバー側で実行する API。 |
| **src/components/** | チャット・プレイヤー・参加者バー・Now Playing 等。`chat/`, `player/`, `room/` で分類。 |
| **src/lib/** | Supabase クライアント、Ably/Socket 接続、videoId 抽出、YouTube oEmbed 等。 |
| **src/hooks/** | useRealtime, useChat, usePlayer 等のカスタムフック。 |
| **src/types/** | メッセージ・セッション・ユーザー等の共通型。 |
| **server/** | Socket.io を自前で立てる場合のみ。Ably の場合は不要。 |
| **docs/** | 既存の企画・仕様書。 |

## 4. 認証・複数ルーム対応時に増やすもの（目安）

- `app/(auth)/login/page.tsx`, `register/page.tsx`
- `app/room/[roomId]/page.tsx`（ルーム別画面）
- `src/lib/supabase.ts`（クライアント）、サーバー用は `app/api` 内で createClient
- `src/components/room/RoomList.tsx`（ルーム一覧・作成入口。登録ユーザーのみ作成可）

---

## 5. 注意

- Next.js 14 で `src/` を使う場合、`app/` を `src/app/` に置く構成も可。その場合は `src/app/`, `src/components/` のように `src` 配下にまとめる。
- 本構成では **app をルート直下**、**components/lib を src 配下** にしている。`create-next-app` で `--src-dir` を付けると `src/app` になるため、どちらかに合わせて揃えること。

---

## 6. 作成済みの初期ファイル（e:\mc 直下で構築する場合）

以下はリポジトリに含まれる初期配置の例。

| パス | 役割 |
|------|------|
| `.env.example` | 環境変数名の一覧。`.env.local` にコピーして値を設定。 |
| `src/types/chat.ts` | チャットメッセージ等の型定義。 |
| `src/lib/youtube.ts` | videoId 抽出・YouTube URL 判定。 |
| `src/lib/ably.ts` | Ably 接続用（プレースホルダー）。 |
| `src/components/chat/Chat.tsx` | チャット表示エリア。 |
| `src/components/chat/ChatInput.tsx` | 入力欄。 |
| `src/components/player/YouTubePlayer.tsx` | YouTube 埋め込み。 |
| `src/components/room/NowPlaying.tsx` | 現在の曲・AI 解説。 |
| `src/components/room/UserBar.tsx` | 参加者バー。 |
| `src/hooks/index.ts` | カスタムフックの出口。 |

**Next.js の app/** は `npx create-next-app@latest . --typescript --tailwind --eslint --app --src-dir` を実行すると `src/app/` として作成される。実行後、`src/app/page.tsx` から上記コンポーネントを import してレイアウトを組む。

**ローカル並行開発**: Music8 を 3001 で動かしているときは、洋楽AIチャットを **3002** で起動する（`package.json` の `dev` を `next dev -p 3002` に設定）。`docs/ローカル並行開発.md` を参照。

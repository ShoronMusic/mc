# 選曲時の処理順（概要）

コード参照なしの手順メモ。実装の詳細は `AGENTS.md` や各 API・コンポーネントを参照。

---

## 1. 選曲直後〜曲解説まで（クライアント起点）

1. 部屋で動画 ID が切り替わる（同期／チャットから URL など）。
2. **選曲アナウンス用**にサーバーへリクエストが飛ぶ（`announce-song`）。
3. **曲解説・曲紹介パック用**にサーバーへリクエストが飛ぶ（`comment-pack`）。同じ動画の「もう一度再生」だけはここを省略することが多い。
4. 上記はブラウザ側ではほぼ同時に始まる（待ち合わせはしない）。

---

## 2. 各サーバールートの中身（announce と comment-pack でそれぞれ同様の型）

5. **YouTube oEmbed** でタイトル・投稿者名（チャンネル表示に相当）を取得する。
6. **YouTube Data API（動画詳細）** で snippet 等を取得する（API キー未設定ならここは取れず、以降は oEmbed 中心になる）。
7. 必要なら **DB の表示上書き**（管理者が直したアーティスト・タイトル）を読む。あれば **タイトル／作者の入力としてはこちらを優先**する。
8. 確定した「タイトル文字列」と「作者名」＋取れた **snippet（説明・チャンネル名など）** を使い、**アーティスト名と曲名を分離・補正**する（有名 PV 固定、環境による簡易モード、曖昧時は MusicBrainz、それ以外はヒューリスティック＋説明欄補完など）。
9. **邦楽扱いか・AI を出すか**などを、分離結果とメタデータで判定する。
10. **announce**：上記の表記で「〇〇さんの選曲です！」＋アーティストと曲の行を組み立てて返す（Gemini は使わない）。
11. **comment-pack**：キャッシュ（過去に保存した解説）があればそれを返し、なければ Gemini で生成して保存する。チャットに出す文の先頭が **[DB]** か **[NEW]** かは、**保存済み再利用かどうか**で決まる。
12. パックが空などのときは **旧来の曲解説 API** に一度だけフォールバックする場合がある。

---

## 3. 選曲者・履歴まわり（並行して動くもの）

13. ログイン済みの**選曲した本人**だけ、**個人の視聴履歴**用 API に記録することがある（部屋の履歴とは別）。
14. **部屋の視聴履歴**への追加は、選曲から**約10秒後**にクライアントから POST される想定。
15. その後、一覧の再取得で**視聴履歴リスト**が更新され、全員の UI が揃う。少し遅れて再読み込みトリガーもかかる。

---

## 4. 視聴履歴パネル内の Music8（メインアーティスト／ソングデータ）

16. 一覧のうち**今再生中の動画 ID と一致する行**を「現在の曲」として使う。
17. その行から（または行のタイトル文字列から）**タブ用のアーティスト名・曲名**を解決する。
18. **music8 のアーティスト用 JSON** にアクセスし、取れたら「メインアーティスト」タブを出せる状態にする。
19. **music8 の曲データ検索**で曲 JSON を取りに行き、取れたら「ソングデータ」タブを出せる状態にする。
20. 履歴行がまだ無い間は「現在行」が無いので、**履歴ベースの Music8 タブは後から**効き始める（POST 反映・再取得後）。

---

## 補足

- **先に YouTube（と必要なら DB 上書き）で文字情報をそろえる → そのあとでアーティスト／曲分離**、という順は **announce も comment-pack も同じ**。
- **Music8** は **曲解説の生成には使わず**、主に**視聴履歴パネルのタブ表示**の段階で使う。

## 運用: 通知メールが届かないとき

- **Good / 悪いだけ**ではメールを送っていない。**吹き出しアイコンからの詳細フィードバック（モーダル送信）**のときだけ Resend 送信する。
- **`RESEND_API_KEY` が無い**（または本番の Vercel 環境変数に未設定）→ メールは送られず、DB のみ保存。サーバーログに `Email send failed` / `RESEND_API_KEY is not set`。
- **`.env.local` を変えたら** Next の開発サーバーを**再起動**する。
- **迷惑メールフォルダ**、`FEEDBACK_TO_EMAIL` の宛先ミスを確認。
- **送信元 `RESEND_FROM`**: 本番では Resend で**ドメイン検証済み**のアドレス推奨。`onboarding@resend.dev` のみだと届きにくい・制限がある場合がある。
- API レスポンス: 詳細FB時は `{ ok: true, emailSent: true|false }`。UI でも `emailSent: false` のとき短い注意を表示する。

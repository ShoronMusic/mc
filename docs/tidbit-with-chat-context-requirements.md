# 豆知識「チャットの流れを見ながら」要件定義

## 1. 目的・背景

- **現状**: 30秒無発言時に披露する豆知識は、**現在流れている曲（タイトル・アーティスト）**だけを AI に渡して生成している。チャットの会話内容は参照していない。
- **目的**: 直近のチャットの流れを渡し、**会話の文脈に合った**豆知識や一言をその場で生成する。同じ曲でも「何の話題で盛り上がったか」に応じて内容を変えられるようにする。

---

## 2. 機能要件

### 2.1 渡す情報（入力）

| 項目 | 内容 | 備考 |
|------|------|------|
| 現在の曲 | これまで通り。YouTube のタイトル・アーティスト（または videoId のみ） | 曲がなければ「洋楽・音楽の豆知識」にフォールバック |
| **直近のチャット** | 直近 N 件のメッセージ（発言者名・本文・種別） | N は 5〜10 件程度を想定。AI のトークン制約と相談しつつ決定 |

### 2.2 AI の振る舞い（プロンプト方針）

- 直近の会話で**出てきたアーティスト・曲・ジャンル・時代・感想**などを把握し、それに**自然につながる**豆知識や一言を出す。
- 会話と無関係な豆知識を突然出さない。話が飛ぶ場合は「洋楽の話ですが」「ところで」などの前置きは既存どおり利用可。
- 豆知識の長さ・トーン（40〜100文字、さりげなく）は現行仕様を維持。

### 2.3 制約・境界

- **件数**: 直近メッセージは**上限を設ける**（例: 最新 8 件）。長すぎるとトークン消費・コスト・レイレンシが増えるため。
- **種別**: ユーザー発言・AI 発言・システム発言のうち、**ユーザーと AI の発言のみ**を渡す（システムは原則除外）。
- **文字数**: 各メッセージ本文を**そのまま**渡すか、**長い場合は省略**するかは実装で決定（推奨: 長文は 100 文字程度で打ち切り）。

---

## 3. 非機能要件

| 項目 | 要件 |
|------|------|
| レイテンシ | 豆知識 1 回の生成は現状と同程度（数秒以内）を目安。チャット履歴追加による遅延は最小限に抑える。 |
| トークン | 直近チャットを渡す分だけプロンプトが増えるため、件数・打ち切りで上限を設け、月次コストに影響が大きくならないようにする。 |
| 後方互換 | チャット履歴が空（初回挨拶だけなど）のときも、現行と同様に「曲」または「洋楽の豆知識」だけで生成できるようにする。 |

---

## 4. 変更対象

### 4.1 クライアント（ルーム）

- 30秒無発言で **tidbit API を呼ぶとき**、**直近のメッセージ一覧**をリクエスト body に含める。
- フォーマット例: `recentMessages: { displayName, body, messageType }[]`（最新が末尾など、順序を明示）。

### 4.2 API（tidbit）

- **POST /api/ai/tidbit**
  - 既存: `body.videoId`（任意）
  - 追加: `body.recentMessages`（任意、配列）。省略時は従来どおり「曲」のみで生成。

### 4.3 サーバー（Gemini）

- **generateTidbit(currentSong, recentMessages?)**
  - 第 2 引数で直近メッセージを受け取る。
  - プロンプトに「直近の会話」を要約またはそのまま含め、「この流れに合った豆知識や一言を」と指示を追加する。

---

## 5. 受け入れ条件

- [ ] 直近チャット（例: 8 件）を渡した場合、生成される豆知識が**会話で出たアーティスト・曲・話題に言及している、または自然につながっている**こと。
- [ ] `recentMessages` を渡さない（または空）の場合、**現行と同様に曲 or 洋楽の豆知識だけで生成**され、エラーにならないこと。
- [ ] 30秒無発言のトリガーや、豆知識の表示タイミング・UI は現行のままであること（変更は「渡すデータ」と「プロンプト」に限定）。

---

## 6. 今後の拡張（スコープ外）

- 豆知識の「テーマ」をユーザーが選べる機能。
- チャット履歴の要約を別 API で事前生成してから tidbit に渡す方式（コスト・レイテンシの最適化）。

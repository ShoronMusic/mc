# 公開Go/No-Goチェックリスト（洋楽AIチャット）

最終更新: 2026-04-14

## 使い方

- 各項目を **Go / 保留 / No-Go** で判定する。
- **No-Go が1つでもあれば本公開は見送る**。
- 保留が3つ以上ある場合は、クローズドβ（招待制）公開までに留める。

## 1. コア体験の成立

- [ ] 初回ユーザーが迷わず「入室 → 再生 → 会話」まで完了できる
- [ ] 主要導線（選曲、AI解説、履歴閲覧、退室）が3分以内に一通り試せる
- [ ] 主要ブラウザで致命的なレイアウト崩れがない（最低: Chrome/Edge）

判定基準:
- Go: 致命的な詰まりなし
- No-Go: 導線途中で進行不能が再現する

## 2. 安定性・エラー復帰

- [ ] API失敗時にユーザー向けの再試行導線がある
- [ ] 画面フリーズや無限ローディングが常態化していない
- [ ] 主要操作でのコンソールエラーが運用許容範囲

判定基準:
- Go: 失敗しても再試行で復帰可能
- No-Go: 再読み込み以外で復帰できない不具合が残る

## 3. AI品質（最低ライン）

- [ ] 明らかな事実誤認・意味不明出力の頻度が許容範囲
- [ ] 不適切/攻撃的/危険な表現を抑制できている
- [ ] 出力が空振りした場合の代替表示（再生成案内等）がある

判定基準:
- Go: 体験を壊す出力が稀
- No-Go: 日常的に不適切または破綻した出力が出る

## 4. モデレーション・安全対策

- [ ] 通報または問題報告の導線が利用可能
- [ ] 禁止行為・利用ルールがユーザーに明示されている
- [ ] 悪用時の運用手順（誰が、いつ、どう対処するか）が決まっている

判定基準:
- Go: 事後対応フローが現実的に回る
- No-Go: 問題発生時の窓口や対処責任が不明確

## 5. 法務・ポリシー表示

- [ ] 利用規約ページが公開され、導線から到達できる
- [ ] プライバシーポリシーが公開され、収集データが明記されている
- [ ] 問い合わせ先（メール/フォーム）がユーザーに提示されている

判定基準:
- Go: 必須ページが公開・参照可能
- No-Go: 規約/プライバシーのどちらかが未整備

## 6. 計測・監視

- [ ] 最低限の行動計測（入室、送信、再生、離脱）が取れている
- [ ] サーバーエラーを検知できる（ログ/通知）
- [ ] 障害時に原因追跡できる情報が残る

判定基準:
- Go: 障害発生時に再現・切り分け可能
- No-Go: 壊れても原因が追えない

## 7. パフォーマンス（体感）

- [ ] 初回表示が著しく遅くない
- [ ] 連続利用時にメモリ/描画負荷で操作不能にならない
- [ ] 主要操作の応答時間がユーザーにとって許容範囲

判定基準:
- Go: ストレスはあるが利用継続できる
- No-Go: 待ち時間やカクつきで離脱が多発する

## 8. 運用体制

- [ ] 問い合わせ対応の担当とSLA目安が決まっている
- [ ] 障害時の告知方法（例: X、告知ページ）が決まっている
- [ ] 緊急停止/機能制限の判断者が明確

判定基準:
- Go: 少人数でも回る運用が定義済み
- No-Go: 障害時の意思決定者が不在

## 9. リリース範囲の明確化

- [ ] 「本公開」か「招待制β」かを明文化している
- [ ] 未実装機能・既知課題をユーザー向けに明示している
- [ ] サポート対象外（環境/機能）を明示している

判定基準:
- Go: 期待値コントロールができている
- No-Go: ユーザー期待と実装実態が大きくズレる

## 10. 収益化導線（任意）

- [ ] 広告導入時の表示面を限定し、体験劣化を検証する計画がある
- [ ] AdSense前提ならポリシー遵守の運用ルールがある
- [ ] 初期は広告なし運用でも継続可能なコスト見通しがある

判定基準:
- Go: 収益化が体験を壊さない設計になっている
- No-Go: 収益化のためにコア体験を犠牲にする状態

# Next.js サイト構築：外部利用サービス一覧

Music8 Lounge（Next.js）で利用する外部サービスの一覧。

---

## 一覧表

| # | サービス名 | 用途 | 必須/任意 | 無料枠 | 備考 |
|---|------------|------|-----------|--------|------|
| 1 | **Vercel** | ホスティング（Next.js のデプロイ） | 必須 | あり（Hobby） | 本番URLの提供。Git 連携で自動デプロイ。 |
| 2 | **Supabase** | データベース（PostgreSQL）・認証（Auth） | 必須 | あり（Free） | ユーザー、セッション、会話ログ、アンケート等を保存。Google/メール認証。 |
| 3 | **Socket.io サーバー** または **Ably** | リアルタイム通信（チャット・同期再生） | 必須 | Ably はあり / Socket は自前サーバー | メッセージ即時配信・再生位置同期。Vercel では Socket サーバー不可のため Ably または別ホストで Socket。 |
| 4 | **Google Gemini API** | AI（曲解説・チャット参加・要約） | 必須 | あり（制限あり） | 楽曲解説生成、会話への相槌・盛り上げ、セッション要約。 |
| 5 | **YouTube IFrame Player API** | 動画再生（埋め込みプレイヤー） | 必須 | 無料 | 同期再生の制御（再生・停止・シーク）。APIキー不要。 |
| 6 | **YouTube oEmbed** または **YouTube Data API v3** | 曲メタデータ取得（タイトル・チャンネル名） | 推奨 | oEmbed は無料 / Data API はクォータ | アーティスト・曲名の取得。MVP は oEmbed のみでも可。 |
| 7 | **ドメイン取得** | 本番用ドメイン（例: lounge.music8.jp） | 任意（MVP 後） | なし | 年額 約1,500円〜。Vercel の *.vercel.app でも開始可。 |

## 各サービスの概要

### 1. Vercel
- **用途**: Next.js アプリのビルド・ホスティング・CDN。
- **契約**: アカウント作成後、Git リポジトリを連携してデプロイ。
- **環境変数**: `GEMINI_API_KEY`、`NEXT_PUBLIC_SUPABASE_URL`、`NEXT_PUBLIC_SUPABASE_ANON_KEY` 等を本番で設定。

### 2. Supabase
- **用途**: PostgreSQL DB、Supabase Auth（Google・メール/パスワード）、必要に応じて Realtime。
- **契約**: プロジェクト作成。URL と anon key を Next.js の環境変数に設定。
- **参照**: `docs/DB設計_会話ログ保存版.md`、`docs/外部DBサービス選定.md`。

### 3. リアルタイム通信
- **Ably**: マネージドサービス。Vercel のみで完結。無料枠あり。チャット・再生位置をチャネルで配信。
- **Socket.io**: 自前で Node サーバーを立てる場合。Railway / Render / Fly.io 等に Socket サーバーをデプロイし、フロントはその URL に接続。

### 4. Google Gemini API
- **用途**: 曲の豆知識・解説（artist + title を入力）、チャットへの相槌・感想、セッション要約。
- **契約**: Google AI Studio で API キー取得。従量課金または無料枠。環境変数 `GEMINI_API_KEY` に設定。

### 5. YouTube IFrame Player API
- **用途**: ページ内に YouTube プレイヤーを埋め込み、再生・一時停止・シークをプログラムで制御。
- **契約**: 不要。script タグで読み込み。利用規約（埋め込みの無加工使用）を遵守。

### 6. 曲メタデータ（YouTube）
- **oEmbed**: `https://www.youtube.com/oembed?url=...` でタイトル・チャンネル名を取得。APIキー不要。
- **YouTube Data API v3**: 動画詳細（説明・タグ・長さ等）を取得。API キーとクォータが必要。MVP では oEmbed のみで可。

### 7. ドメイン
- **用途**: 本番の独自ドメイン。任意。MVP では `*.vercel.app` で公開してよい。

## 認証の選択肢（補足）

| 方式 | サービス | 備考 |
|------|----------|------|
| Supabase Auth | Supabase | Google・メール/パスワード。DB と同一プロジェクトで管理しやすい。 |
| Firebase Auth | Firebase | M8 と「同じ Google アカウント」で揃えたい場合。DB は Supabase のままでも可。 |

→ 新規でまとめるなら **Supabase Auth** を推奨。

---

## まとめ（MVP で必要な外部サービス）

| カテゴリ | サービス |
|----------|----------|
| ホスティング | Vercel |
| DB・認証 | Supabase |
| リアルタイム | Ably または Socket.io（別ホスト） |
| AI | Google Gemini API |
| 動画 | YouTube IFrame Player API |
| 曲メタ | YouTube oEmbed（必須）、Data API v3（任意） |
| ドメイン | 任意（のちに取得可） |

上記を押さえれば、Next.js サイト構築に必要な外部利用サービスは揃う。

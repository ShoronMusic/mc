# ローカル並行開発（Music8 と洋楽AIチャットを同時に動かす）

Music8 を **localhost:3001** で運用している場合、洋楽AIチャット（本プロジェクト）は **別ポート** で起動し、両方を同時に開いて開発できるようにする。

---

## 1. ポートの割り当て

| プロジェクト | ポート | URL |
|--------------|--------|-----|
| **Music8** | 3001 | http://localhost:3001 |
| **洋楽AIチャット**（e:\mc） | **3002** | http://localhost:3002 |

→ 洋楽AIチャットの dev サーバーは **3002** で起動する。

---

## 2. 洋楽AIチャットの起動方法

### 2.1 package.json の dev スクリプトで固定する（推奨）

`package.json` の `scripts.dev` を次のようにする。

```json
"scripts": {
  "dev": "next dev -p 3002",
  "build": "next build",
  "start": "next start",
  "lint": "next lint"
}
```

通常の `npm run dev` で常に 3002 が使われる。

### 2.2 都度指定する

```bash
npm run dev -- -p 3002
# または
npx next dev -p 3002
```

---

## 3. 並行運用の手順

1. **Music8** を起動（例: そのプロジェクトで `npm run dev` → 3001）
2. **洋楽AIチャット** を起動（e:\mc で `npm run dev` → 3002）
3. ブラウザで  
   - Music8: http://localhost:3001  
   - 洋楽AIチャット: http://localhost:3002  
   を開いて同時に開発する。

---

## 4. 注意

- `create-next-app` で本プロジェクトを初期化したあと、**最初に `package.json` の `dev` を `next dev -p 3002` に書き換える**と、以降はポートを気にせず並行運用できる。
- 環境変数（`.env.local`）はプロジェクトごとに別。洋楽AIチャット用の Supabase / Gemini / Ably 等は e:\mc の `.env.local` に設定する。
